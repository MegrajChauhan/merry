# interfaceRet_t

## Overview

When interfaces are used, the operations give following results:

| Value | Name | Description |
| ----- | ---- | ----------- |
| 0 | INTERFACE_FAILURE | The operation on the interface resulted in an unknown failure. |
| 1 | INTERFACE_SUCCESS | The operation on the interface finished successfully. |
| 2 | INTERFACE_TYPE_INVALID | The operation is the meant for the interface type passed. |
| 3 | INTERFACE_HOST_FAILURE | The operation failed because the host failed. |
| 4 | INTERFACE_MISCONFIGURED | The interface is not configured for the said operation. |
| 5 | INTERFACE_INVALID_ARGS | Invalid arguments passed for the operation. |
| 6 | INTERFACE_ACTION_DANGEROUS | **NOT USED**(for now) |

There is also the **interface_t** which represents the interface being used.

| Value | Name | Description |
| ----- | ---- | ----------- |
| 0 | INTERFACE_TYPE_FILE | The interface is a FILE. |
| 1 | INTERFACE_TYPE_PIPE | The interface is a PIPE. |
| 2 | INTERFACE_TYPE_MEM_MAP | The interface is a mapped memory. |
